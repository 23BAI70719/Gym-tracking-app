1. Project Overview

GymTracker is a comprehensive full-stack web application developed using the MERN stack (MongoDB, Express.js, React 18, Node.js) that enables fitness enthusiasts to systematically track their workout progress, monitor physical improvements, and achieve their fitness goals through data-driven insights.

The application addresses a common challenge faced by gym-goers — the lack of a structured, digital system to record and analyze workout data over time. Traditional methods such as paper logs or generic spreadsheets fail to provide meaningful visualizations or personalized insights. GymTracker solves this by offering an intuitive, responsive web interface backed by a secure RESTful API.

The backend follows the MVC (Model-View-Controller) architectural pattern with Express.js handling routing, Mongoose ODM managing MongoDB interactions, and JWT middleware securing protected endpoints. The frontend is built with React 18 using functional components and hooks, with Tailwind CSS providing a modern dark-themed responsive UI.

Key Features
  • Secure Authentication: JWT-based login/signup with bcrypt password hashing (salt rounds: 12)

  • Progress Charts: Weekly (7-day) and monthly (30-day) bar and line charts using Chart.js
  • Dashboard Statistics: Total workouts, weekly count, monthly count, top 5 exercises

  • Goal Tracker: Set weekly workout targets with animated progress bar and insights
  • Workout History: Filter by date range and exercise name, grouped by date
  • Exercise Autocomplete: Smart suggestions while typing exercise names
  • Volume Preview: Live calculation of total volume (sets x reps x weight) while logging
  • Responsive Design: Works seamlessly on mobile, tablet, and desktop

2. Objectives
  • Build a centralized digital platform to log and track gym workout sessions
  • Implement secure JWT-based authentication with bcrypt password hashing
  • Allow users to record exercise name, sets, reps, weight, date, and notes per session
  • Provide interactive weekly and monthly progress charts using Chart.js
  • Display dashboard statistics: total workouts, weekly count, monthly count, top exercises
  • Implement BMI calculator with category classification saved to user profile
  • Allow users to set and track personalized fitness goals with progress visualization
  • Enable workout history filtering by date range and exercise name
  • Protect all workout routes with JWT middleware — user-specific data only
  • Build a responsive, mobile-friendly dark-themed UI using Tailwind CSS
  • Follow MVC architecture and RESTful API design principles throughout
  • Use MongoDB aggregation pipelines for real-time analytics and chart data
Problem Statement

Design and develop a full-stack web application that allows users to:

  • Register and securely authenticate using email and password
  • Log workout sessions with detailed exercise information (sets, reps, weight)
  • View progress through interactive charts and statistics on a dashboard
  • Calculate and track Body Mass Index (BMI) saved to their profile
  • Set and monitor personalized fitness goals with measurable progress
  • Filter and review workout history by date range and exercise type
Motivation
  • Lack of structured tracking: Most gym-goers rely on memory or paper notes which are unreliable
  • No progress visualization: Without charts, it is hard to see improvement trends over weeks/months
  • Goal accountability: Users need a system to set and track fitness goals with measurable progress
  • Health metrics: BMI tracking helps users understand their body composition changes
  • Accessibility: A web-based solution accessible from any device without installation

6. Database Design

GymTracker uses MongoDB database named gym-tracker with 2 collections. All collections are defined through Mongoose ODM with schema validation and indexing.

6.1 users Collection

Stores registered user accounts with profile, BMI, and goal data.

FieldTypeValidationDescription
_idObjectIdAutoMongoDB document ID
nameStringRequired, 2-50 charsFull name of user
emailStringRequired, unique, regexLogin email (lowercase)

goalValueNumberOptionalTarget value for goal
goalExerciseStringOptionalExercise for goal tracking
heightCmNumberOptionalHeight in cm for BMI
weightKgNumberOptionalWeight in kg for BMI

6.2 workouts Collection

Stores individual workout session records. Compound index on userId + date for fast queries.

Index: { userId: 1, date: -1 } — compound index for fast user-specific date-sorted queries.

7. Module Description

7.1 Authentication Module — controllers/authController.js

Handles user registration, login, profile retrieval, and profile updates. Uses JWT for stateless authentication and bcrypt for password security.

FunctionRouteMethodDescription
signup()/api/auth/signupPOSTValidates input, checks duplicate email, hashes password, creates user, returns JWT
login()/api/auth/loginPOSTValidates credentials, compares bcrypt hash, returns JWT + user object
getMe()/api/auth/meGETReturns current user from JWT session
updateProfile()/api/auth/profilePUTUpdates BMI data and fitness goal settings

7.2 Workout Module — controllers/workoutController.js

Handles all workout CRUD operations and statistics generation using MongoDB aggregation pipelines.

FunctionRouteMethodDescription
createWorkout()/api/workoutsPOSTValidates input, creates workout linked to userId
getWorkouts()/api/workoutsGETReturns user workouts with date/exercise filters and pagination
deleteWorkout()/api/workouts/:idDELETEVerifies ownership, deletes workout by ID
getStats()/api/workouts/statsGETAggregates total, weekly, monthly counts + chart data + top exercises

7.3 Auth Middleware — middleware/auth.js

Protects all workout routes by verifying the JWT token from the Authorization header.

CheckResponse
